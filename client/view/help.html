WellInverter help pages

Operations on data files

WellInverter supports Microsoft Excel XLSX data files generated by Tecan microplate readers.

When reading a data file, WellInverter uses the following constraints:

  • Only the first worksheet in a Tecan workbook is read.
  • Each data table is preceded with the name of the measurement type.
  • Only two types of measurement are allowed: absorbance and fluorescence.
  • For an absorbance data table, the name of the measurement type before the measures table starts with "abs".
  • For a fluorescence data table, the name of the measurement type before the data table starts with "fluo".
  • The 1st row of a data table has the following format: the 1st column cell is labelled "Cycle Nr.". Subsequent columns are the time points from 1 to the number of time points.
  • The 2nd row of a data table has the following format: the 1st column cell is labelled "Temp [Â°C]". Subsequent columns contain the temperature at the different time points.
  • From the 3rd row onwards, rows have the following format. For odd row numbers, the 1st column cell is the name of a Well, ordered from "A1" to "H12", and subsequent columns contain the values of the measurement for the well at the different time points. For even row numbers, the 1st column cell is "Time[ms]", and subsequent columns contain the different time points, expressed in ms.

Allowed characters in the name of a Tecan experiment file are: a-z, A-Z, 0-9, _ and -.

Storage of experimental data

The data analyzed by means of WellInverter is not stored locally on your computer. You have to upload the data to the WellInverter server before starting the analysis. Once an experiment has been uploaded, the data contained in the file is stored on the server. All operations carried out on the data (e.g., outlier detection) and the parameters of the analysis (e.g., half-live of fluorescent protein) are also stored on the server.

WellInverter uses the JSON (JavaScript Object Notation) format to store the data associated with an experiment.

While experiments are stored on the server, they can be downloaded at any time. It is advised to do this on a regular basis, since like in any computer, the hard disk of the WellInverter may crash.

Download an experiment

Once an experiment is uploaded on the server, you can download it anytime.

To download an experiment:

1. Right-click on the experiment node, and select Download WellInverter (JSON) experiment file.

Delete an experiment

Before you delete an experiment, you should download it so as to be able to restore it, if necessary.

To delete an experiment:

1. Right-click on the experiment node to delete, and select Delete experiment.

Rename an experiment

To rename an experiment:

1. Right-click on the experiment node to rename, and select Rename experiment.

2. Modify the name of the experiment. Allowed characters in an experiment name are: a-z, A-Z, 0-9, _ and -.

3. Type Enter to validate the change.

Export analyzed data

Even though WellInverter has a range of analysis and plotting capabilities, one may sometimes want to export (partially) treated data in order to continue the analysis with another program.

To export data:

1. Right-click on the experiment node, and select Export analyzed data. Only data that have been analyzed in the current session are exported. When exporting data, a ZIP file containing text files associated with the primary data, the background-corrected data, and the quantities computed from the data (growth rate, promoter activity, protein concentration) are sent to the browser.

Experiment info.

When uploading a Tecan data file, metadata about the experiment such as the start time of the experiment and the settings of the instrument are stored in the JSON experiment file

Parameters

There exist various parameters associated with the different analysis steps of an experiment.

These parameters are grouped in a form.

To open the parameter form:

1. Click on the Parameters node of the current experiment.

There are three categories of parameters:

  1. Outlier detection.These parameters specify how outliers are detected and eliminated from the data set before the analysis. There is one set of outlier detection parameters for each type of measurement (typically absorbance and fluorescence).
  2. Default background. You can set a default background level for each type of measurement. These default values are overruled when a background well is defined for a given well and measurement type.
  3. Synchronization of wells. The time axis of each of the wells is shifted in such a way that the cross correlation between the data in the wells of the microplate is maximized. You can specify a maximum value for the shift. Notice that a maximum shift equal to 0 disables synchronization.

Hovering ? button with the mouse displays help for the corresponding parameter.

Background definition

To define background levels for a set of wells and for the different types of measurements:

  1. Select the type of measurement for which you want to define the background, in the Background subtree.
  2. Select the wells for which you want to define the background: click on a well, a column or row identifier, or draw a rectangular selection with the mouse. If a background has already been defined for a selected well, it will be replaced.
  3. Click on the Set background button.
  4. Click on the well serving as the background for the selected wells.

Once a background has been defined for a well or set of wells, the background definition window appears as follows:

Background wells are displayed with a green circle. Wells that have a background are displayed with the name the background well. For instance, in the above screenshot, well E1 has A1 as background.

To clear the background definition for a set of wells:

  1. Select the measurement type in the Experiments tree for which you want to clear the background.
  2. Select the wells for which you want to clear the background: click on a well, a column name or a line name, or draw a rectangular selection.
  3. Click on the Clear button.

Notice that, if no background has been defined for the wells, the default background specified in the Parameters page will be used for background correction.

Outlier detection

To detect and remove outliers:

1. Select the type of measurement for which you want to detect outliers, in the Outliers subtree.

2. Select the well for which you want to detect outliers, by clicking on a well in the graphical representation of the microplate, in the lower left corner of the screen

Next, two methods exist for detecting outliers: a manual and an automated method.

a. To define outliers manually:

3.a Check Show trend, if it is unchecked.

4.a Uncheck Hide outliers, if it is checked.

5.a Indicate the trend in the data on the plot by clicking at regular distances. This trend serves as a reference for detecting outliers: data points that are farther than a certain tolerance removed from the trend are considered outliers. The black diamond-shaped points that indicate the trend in the data can be moved by dragging them with the mouse. They can be removed by clicking on the plot, in the neighborhood of the point. To add an additional point between two existing points indicating the trend, click between these two points, but not close to the segment connecting them (otherwise, this operation will remove the nearest black diamond-shaped point), and then drag the added point to the desired position.
Once a trend has been defined the outlier detection window is displayed as follows:

6.a Set the tolerance in the Tolerance text box. You can also use the upward and downward arrows in the text box to change the current tolerance value.

7.a Click on the Filter button to remove outliers from data set, that is, the data points that lie at a distance larger than the tolerance from the indicated trend.

b. To define outliers automatically:

3.b Click on the Detect button. Note that the automated detection of outliers is additive: when this operation is applied to a data set from which some outliers have already been removed by the operations in 3.a, these outliers are not considered as data points by the automated outlier detection algorithm. The parameters characterizing the automated outlier detection algorithm are specified in the Parameters page.

Plots

To draw a plot of a given measure:

1. Click on the Plots node in the experiments tree.

2. Select one or several wells for which you want to plot the data or the quantities derived from the data.

3. Choose the type of measurement and plot the profiles of interest associated with these measurements. The profiles may concern the primary or corrected data and the growth rate, promoter activity, and protein concentration. Use + and - buttons to add or remove plots. The types of measurements are those defined in the uploaded data file. The types of profiles that can be plotted depend on the type of measurement. Notice that growth rate can only be plotted for absorbance-type measurements and promoter activities and protein concentrations are associated with fluorescence-type measurements. (although they also require absorbance-type measurements to be computed).

4. In addition to plotting profiles for individual wells, the mean and the standard error of the mean of the selected set of wells can be plotted by checking the corresponding checkboxes.

Plot color

The color of the plotted profiles can be controlled either manually of automatically. With manual color control, once a color is selected by clicking in the color palette (see figure below), it is used for plotting all subsequently selected wells, until another color or automatic control is selected. With automatic color control, WellInverter chooses a new color in the palette for each new plot. Note that color control does not apply to plots of the mean and the standard error of the mean of a selected group of wells. These profiles always appear in black.

Axis scale

The measurements are plotted on either a linear or a logarithmic Y-axis scale.

To choose the type of scale:

1. Select the appropriate scale type at the top of left and/or right Y-axis.
